## 一、选型原则

| 原则 | 说明 |
|------|------|
| **合适优于热门** | 选择最适合项目需求的技术，而非最流行的 |
| **简单优于复杂** | 优先选择开箱即用、配置简单的方案 |
| **稳定优于新颖** | 优先选择成熟稳定、社区活跃的技术 |
| **轻量优于重量** | 项目定位轻度休闲游戏，避免过重的技术栈 |

## 二、核心技术栈

### 2.1 总览

| 层级 | 技术选型 | 版本要求 |
|------|----------|----------|
| **游戏引擎** | Phaser 3 | ^3.70.0 |
| **开发语言** | TypeScript | ^5.0.0 |
| **构建工具** | Vite | ^5.0.0 |
| **包管理器** | pnpm | ^8.0.0 |
| **部署平台** | Vercel / Netlify / GitHub Pages | - |

### 2.2 选型理由

#### 游戏引擎：Phaser 3

| 对比项 | Phaser 3 | 纯 Canvas | PixiJS | Cocos Creator |
|--------|----------|-----------|--------|---------------|
| 开发效率 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 学习曲线 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 打包体积 | ~1MB | 0 | ~250KB | ~10MB+ |
| 游戏功能 | 完整 | 需手写 | 仅渲染 | 完整 |
| 适合本项目 | ✅ | ❌ | ❌ | ❌ |

**核心优势：**
- 专为2D游戏设计，内置精灵/动画/物理/碰撞/音频系统
- WebGL优先渲染，自动降级Canvas，兼容性好
- 社区活跃，文档完善，示例丰富
- 打包体积适中，适合轻度游戏
- 静态部署友好，无需后端

#### 开发语言：TypeScript

| 优势 | 说明 |
|------|------|
| 类型安全 | 编译期发现错误，减少运行时bug |
| IDE支持 | VSCode智能提示、重构、跳转 |
| 代码可维护性 | 接口定义清晰，团队协作友好 |
| 游戏开发友好 | Phaser官方TypeScript定义完善 |

**本项目涉及复杂类型：**
- 5种兵种 + 6种敌人 = 大量实体类
- 状态机、波次系统、战斗系统 = 复杂逻辑
- TypeScript能有效管理这些复杂度

#### 构建工具：Vite

| 对比项 | Vite | Webpack |
|--------|------|---------|
| 开发服务器启动 | 毫秒级 | 秒级 |
| 热更新速度 | 极快 | 较慢 |
| 配置复杂度 | 低 | 高 |
| 打包体积 | 小 | 较大 |

**核心优势：**
- 原生ESM，开发服务器即时启动
- HMR体验极佳，游戏开发调试效率高
- 配置简单，开箱即用

#### 包管理器：pnpm

- 磁盘空间节省（硬链接机制）
- 安装速度快
- 依赖结构清晰，避免幽灵依赖

## 四、核心模块设计

### 4.1 场景流转

```
BootScene → MenuScene → LevelSelectScene → BattleScene → ResultScene
                ↑                                      ↓
                └──────────────────────────────────────┘
```

| 场景 | 职责 |
|------|------|
| BootScene | 加载资源、显示进度条、初始化游戏 |
| MenuScene | 主菜单、开始游戏、成就入口 |
| LevelSelectScene | 关卡选择、显示解锁状态 |
| BattleScene | 战斗核心逻辑、兵种放置、敌人生成 |
| ResultScene | 结算界面、战绩展示、下一关入口 |

### 4.2 实体系统

```typescript
interface Entity {
  id: string;
  x: number;
  y: number;
  hp: number;
  maxHp: number;
  speed: number;
  sprite: Phaser.GameObjects.Sprite;
  update(time: number, delta: number): void;
  takeDamage(amount: number): void;
  destroy(): void;
}

interface Unit extends Entity {
  cost: number;
  attackDamage: number;
  attackRange: number;
  attackSpeed: number;
  target: Enemy | null;
  state: 'idle' | 'moving' | 'attacking' | 'dead';
}

interface Enemy extends Entity {
  reward: number;
  attackDamage: number;
  attackRange: number;
  attackSpeed: number;
  target: Unit | null;
  state: 'moving' | 'attacking' | 'dead';
}
```

### 4.3 战斗系统

```typescript
class CombatSystem {
  checkCollisions(): void;
  calculateDamage(attacker: Entity, defender: Entity): number;
  applySpecialMechanisms(attacker: Entity, defender: Entity): void;
  handleDeath(entity: Entity): void;
}
```

### 4.4 波次系统

```typescript
interface Wave {
  enemies: {
    type: string;
    count: number;
    delay: number;
  }[];
  spawnPoints: { x: number; y: number }[];
}

class WaveSystem {
  currentWave: number;
  totalWaves: number;
  waves: Wave[];
  
  startNextWave(): void;
  spawnEnemy(type: string, x: number, y: number): void;
  isWaveComplete(): boolean;
}
```

## 五、性能优化策略

### 5.1 对象池（Object Pool）

```typescript
class ObjectPool<T> {
  private pool: T[] = [];
  
  get(): T;
  release(obj: T): void;
  preallocate(count: number): void;
}
```

**应用场景：**
- 子弹/投射物
- 特效粒子
- 伤害数字

### 5.2 渲染优化

| 策略 | 说明 |
|------|------|
| Sprite合批 | 使用TexturePacker合并精灵图 |
| 视口裁剪 | 只渲染可见区域 |
| 减少Draw Call | 合并相同纹理的精灵 |
| 关闭抗锯齿 | 像素风格无需抗锯齿 |

### 5.3 内存管理

- 及时销毁离开屏幕的实体
- 复用对象而非频繁创建
- 控制同屏最大实体数量

## 六、资源管理

### 6.1 精灵图规范

| 类型 | 尺寸 | 格式 |
|------|------|------|
| 兵种/敌人 | 64x64 | PNG (透明) |
| 特效 | 128x128 | PNG (透明) |
| UI元素 | 按需 | PNG |
| 背景 | 1920x1080 | JPG |

### 6.2 音频规范

| 类型 | 格式 | 说明 |
|------|------|------|
| 音效 | MP3 / OGG | 短促，< 1秒 |
| 背景音乐 | MP3 / OGG | 循环播放 |

### 6.3 加载策略

- BootScene加载核心资源（进度条、Logo）
- MenuScene预加载常用资源
- BattleScene按关卡加载特定资源

## 七、开发规范

### 7.1 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| 文件名 | PascalCase | `ShieldBearer.ts` |
| 类名 | PascalCase | `class ShieldBearer` |
| 接口名 | PascalCase + I前缀(可选) | `interface Entity` |
| 函数名 | camelCase | `takeDamage()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_ENEMIES` |
| 私有属性 | _前缀 | `private _hp: number` |

### 7.2 代码风格

- 使用ESLint + Prettier
- 遵循TypeScript官方风格指南
- 单文件不超过500行
- 函数不超过50行

## 八、部署方案

### 8.1 构建命令

```bash

pnpm dev

# 构建
pnpm build

# 预览构建结果
pnpm preview
```

### 8.2 部署平台对比

| 平台 | 优点 | 缺点 |
|------|------|------|
| **Vercel** | 部署简单、自动CI/CD、免费额度充足 | 国内访问可能较慢 |
| **Netlify** | 功能丰富、表单处理、免费SSL | 构建时间限制 |
| **GitHub Pages** | 完全免费、与GitHub集成 | 需要手动配置 |

**推荐：Vercel**（部署最简单，自动CI/CD）

### 8.3 部署配置

```json
{
  "buildCommand": "pnpm build",
  "outputDirectory": "dist",
  "framework": "vite"
}
```
