# 戚继光抗倭塔防 - 开发路线图

*文档版本：v1.0 | 更新日期：2026-04-18*

---

## 一、里程碑总览

| 里程碑 | 名称 | 目标 | 核心产出 |
|:-------|:-----|:-----|:---------|
| **M1** | MVP | 验证核心玩法可行性 | 第1关可玩，2兵种2敌人 |
| **M2** | 核心完整 | 完整战斗系统 | 5兵种6敌人，第1-4关 |
| **M3** | 完整版 | 全关卡+UI | 9关完整，完整UI |
| **M4** | 打磨优化 | 性能+体验 | 优化、音效、平衡 |

## 二、依赖关系图

```
项目初始化
    │
    ├──→ 基础配置 (gameConfig.ts)
    │         │
    │         └──→ 类型定义 (types/)
    │                   │
    │                   └──→ Entity基类
    │                             │
    │                             ├──→ Unit基类 ──→ 5种兵种
    │                             │
    │                             └──→ Enemy基类 ──→ 6种敌人
    │
    ├──→ 场景框架
    │         │
    │         ├──→ BootScene (资源加载)
    │         ├──→ MenuScene (主菜单)
    │         ├──→ LevelSelectScene (选关)
    │         ├──→ BattleScene (战斗核心)
    │         └──→ ResultScene (结算)
    │
    └──→ 系统层
              │
              ├──→ ResourceSystem (军饷)
              ├──→ CombatSystem (战斗) ←── 实体系统
              ├──→ WaveSystem (波次) ←── 敌人 + 战斗
              └──→ SpawnSystem (生成) ←── 波次
                        │
                        └──→ 关卡系统 (9关配置)
                                  │
                                  └──→ 完整游戏
```

## 三、Milestone 1: MVP（最小可行性产品）

**目标**：验证核心玩法可行性，第1关可玩

### 3.1 项目搭建

- [ ] 初始化 Vite + TypeScript 项目
- [ ] 安装 Phaser 3 依赖
- [ ] 配置 `tsconfig.json`
- [ ] 配置 `vite.config.ts`
- [ ] 创建目录结构
- [ ] 创建入口 `index.html` + `main.ts`

**提示词**：
```
初始化戚继光抗倭塔防项目：
- 使用 pnpm + Vite + TypeScript
- 安装 phaser@^3.70.0
- 按照技术文档的目录结构创建文件夹
- 配置 tsconfig 和 vite.config
```

### 3.2 基础配置与类型

- [ ] 创建 `src/config/gameConfig.ts` - Phaser游戏配置
- [ ] 创建 `src/types/index.ts` - 核心类型定义
- [ ] 创建 `src/utils/Constants.ts` - 常量定义

**提示词**：
```
开发基础配置模块：
- gameConfig.ts: Phaser游戏配置（800x600分辨率，物理引擎Arcade）
- types/index.ts: Entity、Unit、Enemy接口定义，状态类型
- Constants.ts: 游戏常量（网格大小、移动速度等）
```

### 3.3 场景框架

- [ ] 创建 `src/scenes/BootScene.ts` - 启动场景（资源加载+进度条）
- [ ] 创建 `src/scenes/MenuScene.ts` - 主菜单场景
- [ ] 创建 `src/scenes/LevelSelectScene.ts` - 选关场景
- [ ] 创建 `src/scenes/BattleScene.ts` - 战斗场景（空框架）
- [ ] 创建 `src/scenes/ResultScene.ts` - 结算场景
- [ ] 实现场景流转逻辑

**提示词**：
```
开发Phaser场景框架：
- BootScene: 加载资源，显示进度条，完成后跳转MenuScene
- MenuScene: 主菜单，"开始游戏"按钮跳转LevelSelectScene
- LevelSelectScene: 关卡列表，点击关卡进入BattleScene
- BattleScene: 战斗场景框架，预留update方法
- ResultScene: 显示胜利/失败，返回按钮
- 在gameConfig中注册所有场景
```

### 3.4 实体基类

- [ ] 创建 `src/entities/base/Entity.ts` - 实体基类
- [ ] 创建 `src/entities/base/Unit.ts` - 我方单位基类
- [ ] 创建 `src/entities/base/Enemy.ts` - 敌方单位基类

**提示词**：
```
开发实体基类：
```

**提示词**：
```

- 藤牌兵: 碰撞体积阻挡，最多同时阻挡3个敌人

- 长枪倭寇: 攻击可穿透前排命中后排
- 藤甲倭寇: 接收火器伤害时使用固定值30而非秒杀
- 狙击手: 目标选择按Y坐标排序，选最小（最靠前）
- BOSS: 每12秒计时器触发召唤，周围敌人计数影响攻速

```

```
3. 场景框架（空场景流转）
4. Entity/Unit/Enemy 基类
5. 藤牌兵 + 长枪兵
6. 浪人倭寇 + 倭弓手
7. 战斗系统基础
8. 资源系统
9. 简易HUD
10. 第1关（MVP完成）
```
